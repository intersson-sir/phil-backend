# Где что запускать: фронт и бэкенд

## Два проекта — два места для команд

| Что | Папка проекта | Где вводить команды |
|-----|----------------|---------------------|
| **Фронт** (Next.js, логин, интерфейс) | `phil-frontend` | Терминал в окне/проекте **фронта** (или отдельный терминал, но `cd` в папку фронта). |
| **Бэкенд** (Django API, логин по API) | `phil-backend/phil-backend` | Терминал в окне/проекте **бэкенда** (или отдельный терминал, но `cd` в папку бэкенда). |

Команды `docker compose`, `python manage.py` — **только в папке бэкенда**.  
Команды `npm run dev`, `npm install` — **только в папке фронта**.

---

## Как у тебя сейчас

- Фронт открыт в одном окне (PHIL-FRONTEND), логин на `http://localhost:3000/login`.
- Бэкенд — в другом окне (phil-backend).

Чтобы «Войти» работало, **бэкенд должен быть запущен** на `http://localhost:8000`. Фронт к нему ходит по `NEXT_PUBLIC_API_URL` (по умолчанию `http://localhost:8000`).

---

## Что сделать по шагам

### 1. Запустить бэкенд (в папке бэкенда)

Открой терминал (в Cursor в проекте **phil-backend** или обычный Terminal.app) и выполни:

```bash
cd /Users/andrey/phil-backend/phil-backend
DATABASE_URL="sqlite:///db.sqlite3" .venv/bin/python manage.py runserver
```

Оставь этот терминал открытым — сервер должен работать. В консоли будет что-то вроде: `Starting development server at http://127.0.0.1:8000/`.

### 2. Фронт уже запущен

У тебя уже открыт фронт на `http://localhost:3000/login`. Ничего перезапускать не нужно.

### 3. Проверить адрес бэкенда на фронте

В проекте **phil-frontend** в файле `.env` или `.env.local` должно быть (если хочешь ходить на локальный бэкенд):

```env
NEXT_PUBLIC_API_URL=http://localhost:8000
```

Если этого нет — добавь и перезапусти фронт (`npm run dev` в папке фронта).

### 4. Войти на фронте

На странице входа введи:

- **Логин или Email:** `phil_demo`
- **Пароль:** `PhilDemo2026`

Нажми «Войти». Запрос уйдёт на бэкенд с шага 1; пользователь уже создан в локальной БД.
